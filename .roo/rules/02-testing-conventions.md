# Test Case Rules

## Mandatory Reference Documentation

Before writing, modifying, or analyzing any test cases, you MUST read the following file in its entirety:
`docs/testing-guide.md`

## Core Principles

1. **Single Source of Truth (SSOT)**: The `docs/testing-guide.md` file is the absolute authority for all testing conventions, naming standards, and architectural patterns within this project.
2. **Mandatory Sequence**: Reading this document is a required first step. You must not generate or propose any test implementation until you have aligned with the specific guidelines provided in the guide.
3. **Strict Adherence**: Do not rely on general testing knowledge or external standards if they conflict with the rules defined in `docs/testing-guide.md`.
